System Overview

What is Credify 2026?

A production-ready blockchain-based verifiable credential system specifically designed for academic transcripts. Universities issue tamper-proof digital credentials that students can share selectively with employers while maintaining privacy and cryptographic authenticity.

Key Features:
  • W3C Verifiable Credentials Standard
  • RSA-2048 Digital Signatures
  • Blockchain-based Integrity
  • Credential Versioning System
  • IPFS Distributed Storage
  • Selective Disclosure Privacy
  • Tamper-proof Verification
  • Zero-Knowledge Proofs

System Setup

Prerequisites:
  • Python 3.10+ (Python 3.11 recommended)
  • VS Code or any code editor
  • Git for version control
  • Internet connection for IPFS (optional - has local fallback)
Installation Steps:
1
Clone the Repository
git clone <repository-url>
cd blockchain-credentials
2
Create Virtual Environment
python -m venv venv

# Windows PowerShell
venv\Scripts\Activate.ps1

# Mac/Linux
source venv/bin/activate
3
Install Dependencies
pip install -r requirements.txt
4
Initialize Database
python -c "from app.models import init_database; from app.app import app; init_database(app)"
5
Create Admin User
python scripts/create_admin.py
6
Run the Application
python main.py
Success! Application running at http://localhost:5000
Daily Usage: Activate venv and run python main.py

Issuer Guide

Role: Academic Institution

As an academic institution, you can issue verifiable credentials for student transcripts with full cryptographic security.

Step-by-Step Process:
1
Login as Issuer

Navigate to the login page and authenticate with your issuer credentials

2
Navigate to Issuer Portal

Click on "Issuer" in the navigation menu or visit /issuer

3
Fill Student Information
  • Student Name (required)
  • Roll Number (required)
  • Degree Program (required)
  • University Name (pre-filled)
  • GPA/CGPA (required, 0-10 scale)
  • Graduation Year (required)
  • Semester & Year (optional)
  • Courses (optional, comma-separated)
  • Backlogs & Conduct (optional)
4
Issue Credential

Click "Issue Credential" button. The system will:

  • Create a W3C-compliant verifiable credential
  • Apply RSA-2048 cryptographic signature
  • Store full credential on IPFS network
  • Record credential hash on blockchain
  • Generate unique Credential ID
  • Auto-create student account (if new)
5
Review Credential Details

After issuance, you'll receive:

  • Credential ID: Unique identifier
  • IPFS CID: Content identifier for distributed storage
  • Blockchain Hash: Transaction hash on blockchain
  • Block Number: Position in blockchain
6
Manage Credentials

Use the "Manage Credentials" section to:

  • View all issued credentials
  • Search by student roll number
  • Revoke credentials if needed
  • Create new versions for updates
  • View credential history
Important: Once issued, credentials are immutably recorded on the blockchain. Ensure all information is accurate before issuing. For corrections, use the "Create New Version" feature.
Student Access: Students can login using their roll number as username and password (default: roll number). They should change their password after first login.

Student Guide

Role: Student (Credential Holder)

As a student, you can view your credentials and create privacy-preserving selective disclosures for sharing with employers.

Managing Your Credentials:
1
Login to Student Portal

Use your roll number as username and password (change password after first login)

2
View Your Credentials

Navigate to "Student" portal to see all your issued credentials with details like:

  • Credential ID
  • Degree Program
  • GPA/CGPA
  • Issue Date
  • Status (Active/Revoked/Superseded)
3
View Full Credential Details

Click "View" to see complete credential information including:

  • All academic details
  • Courses completed
  • Blockchain verification status
  • IPFS storage details
  • Cryptographic signature
4
Create Selective Disclosure

Click "Share" to create a privacy-preserving disclosure:

  • Select which fields to share (e.g., only GPA)
  • System generates cryptographic proof
  • Copy the JSON proof to share with employers
  • Hidden fields remain completely private
5
Generate Zero-Knowledge Proofs

Create proofs for:

  • Range Proofs: "My GPA is greater than 7.5" (without revealing exact value)
  • Membership Proofs: "I completed Data Structures" (without revealing all courses)
6
Download PDF Transcript

Download a printable PDF version of your credential with QR code for verification

7
Raise Support Tickets

If you have issues or need corrections, raise a support ticket directly from your dashboard

Privacy Benefits:
What You Can Share Selectively:
  • GPA/CGPA only
  • Degree program only
  • Graduation year only
  • Specific courses only
  • Any custom combination
Privacy Protection:
  • Hide personal details
  • Share only relevant information
  • Maintain full authenticity
  • Cryptographic proof included
  • Zero-knowledge verification
Data Control: You have complete control over what information you share. Employers can only see what you explicitly choose to disclose.

Verifier Guide

Role: Employer/Verifier

As an employer, you can verify the authenticity of academic credentials and selective disclosure proofs with cryptographic certainty.

Verification Methods:
Method 1: Full Credential
  1. Get credential ID from student
  2. Visit verifier portal (no login required)
  3. Enter credential ID
  4. Click "Verify Credential"
  5. Review complete transcript
Method 2: Selective Disclosure
  1. Get disclosure proof from student
  2. Visit verifier portal
  3. Paste JSON proof
  4. Click "Verify Proof"
  5. Review disclosed fields only
Method 3: Zero-Knowledge Proof
  1. Get ZKP from student
  2. Visit verifier portal
  3. Go to "ZKP Verification" tab
  4. Paste ZKP JSON
  5. Verify claims without seeing data
Verification Checks Performed:
  • Blockchain Integrity ✓
  • RSA-2048 Digital Signature ✓
  • SHA-256 Hash Validation ✓
  • Tamper Detection ✓
  • Issuer Verification ✓
  • Revocation Status Check ✓
Verification Results Include:
  • Validity Status: Valid/Invalid/Revoked/Superseded
  • Student Information: Name, degree, GPA, etc.
  • Issuer Details: University name and credentials
  • Issue Date: When credential was created
  • Blockchain Proof: Block number and transaction hash
  • IPFS Storage: Content ID and accessibility status
  • Cryptographic Signature: RSA verification result
Trust Guarantee: Selective disclosure proofs are cryptographically verifiable and maintain the same level of trust as full credentials while protecting student privacy through zero-knowledge proofs.
No Registration Required: The verifier portal is publicly accessible. Anyone can verify credentials without creating an account.

Troubleshooting

Common Issues & Solutions:

Problem: Error when running database initialization command.

Solution:

  1. Ensure virtual environment is activated
  2. Check if instance/ folder exists, create if not
  3. Delete existing instance/credentials.db if corrupted
  4. Run: python -c "from app.models import init_database; from app.app import app; init_database(app)"

Problem: Cannot connect to IPFS network.

Solution:

  • Option 1: System automatically falls back to local storage
  • Option 2: Install IPFS Desktop from ipfs.io
  • Option 3: Use Infura IPFS endpoint (configure in config.py)

Note: Local fallback works perfectly for development and testing.

Problem: Cannot login with created credentials.

Solution:

  • For Issuers: Run python scripts/create_admin.py again
  • For Students: Default password is roll number. Contact admin if locked out
  • Check if database was properly initialized
  • Verify username is correct (roll number for students)

Problem: Credential issuance takes too long.

Solution:

  1. Open app/blockchain/blockchain.py
  2. Reduce difficulty level (default: 4 leading zeros)
  3. Change DIFFICULTY = 4 to DIFFICULTY = 2
  4. Restart application

Note: Lower difficulty = faster mining but less security (fine for development).

Problem: Python cannot find application modules.

Solution:

# Reinstall dependencies
pip install --upgrade pip
pip uninstall -r requirements.txt -y
pip install -r requirements.txt

# Or recreate virtual environment
deactivate
rm -rf venv  # Windows: rmdir /s venv
python -m venv venv
venv\Scripts\activate  # Windows
pip install -r requirements.txt

Problem: Valid credentials show as invalid.

Possible Causes:

  • Blockchain corruption: Delete blockchain_data/ and restart
  • Key mismatch: Issuer keys changed after credential issuance
  • IPFS storage: Credential not found on IPFS (check local storage)
  • Database sync: Database and blockchain out of sync

Fix: For development, delete all data and restart fresh.

Problem: Cannot start Flask app on port 5000.

Solution 1 - Kill Existing Process:

# Windows
netstat -ano | findstr :5000
taskkill /PID [process_id] /F

# Mac/Linux
lsof -ti:5000 | xargs kill -9

Solution 2 - Use Different Port:

python main.py --port 5001
