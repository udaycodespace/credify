The Elite Protocol

Credify 2026 is an elite digital credentialing system. We help universities issue tamper-proof transcripts using blockchain, give students private control over their records, and allow employers to instantly verify authenticity with a single click.

By leveraging Private Blockchain clusters and IPFS decentralised storage, we've eliminated the possibility of academic fraud while ensuring students have 100% ownership of their educational identity.

Environment Injection

Follow the standard sequence to initialize the Credify node on your local machine:

# 1. Clone & Enter Core
git clone https://github.com/udaycodespace/credify.git
cd credify

# 2. Virtual Layer Setup
python -m venv venv
source venv/Scripts/activate # Windows: venv\Scripts\activate

# 3. Dependencies
pip install -r requirements.txt
Security Config (.env)

You MUST configure MAIL_SERVER and MAIL_PASSWORD to enable the MFA activation emails and security alerts.

Institution

University Operations

  • 1. Authority Onboarding

    Institutions log in via the elite portal. System enforces mandatory Time-based One-Time Password (TOTP) setup via Google Authenticator for all issuing officers.

  • 2. Credential Anchoring

    Input Roll Number and GPA. The core generates a signed JSON-LD manifest, uploads to IPFS Cluster, and records the transaction hash on the Private Blockchain.

  • 3. Student Handover

    The system automatically triggers a cryptographically signed activation link to the student's institutional email address.
